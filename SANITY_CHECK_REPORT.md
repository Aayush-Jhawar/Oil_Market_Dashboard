# Full Prediction Pipeline Sanity Check

**Date:** 2026-06-08
**Auditor:** Antigravity AI
**Scope:** Data, Feature, Model, Signal, and Dashboard layers.

## 1. Executive Summary
An end-to-end audit of the prediction system revealed multiple structural issues, including critical data leakage risks, scaling mismatches, and dependency failures. Several fixes have already been deployed during Phase 5 Execution.

## 2. Findings & Recommended Fixes

### A. Regime Engine Outputting `NEUTRAL` (Critical)
*   **Issue:** The `hmmlearn` package requires C++ build tools, which are missing. The engine gracefully degraded into a hardcoded fallback that outputted `NEUTRAL`. Furthermore, the Rule-based classifier was being passed *scaled* features instead of *raw dollar* spreads, making its absolute thresholds fail.
*   **Status:** **FIXED.** Replaced the HMM dependency with the `RuleBasedRegimeClassifier` and fixed the feature passing logic so raw unscaled dollars are sent to the threshold engine.

### B. Look-Ahead Bias in Walk-Forward Validation (Critical)
*   **Issue:** In `walk_forward.py`, the `FeatureTransformer` (which applies StandardScaling and Imputation) was being `fit` on the entire historical dataset before the sliding window began. This means the mean/std for scaling data in 2015 included knowledge of prices from 2024.
*   **Status:** **FIXED.** The `FeatureTransformer` must be instantiated and `fit` exclusively on the `train_data` slice within each window of the loop, then used to `transform` the `test_data` slice.

### C. Missing Target/Current Price Defaults (Medium)
*   **Issue:** If `realized_vol_20d` is missing, `signal_generator.py` defaults to `25%` Volatility. This heavily influences the target/stop placement. In the simulated historical environment, WTI volatility spikes well above 25%, causing the targets to sometimes look detached from local price action.
*   **Fix:** Introduce dynamic ATR fallback calculations. If `realized_vol_20d` is NaN, calculate it dynamically from the `closes` array rather than hardcoding `25`.

### D. Target Scaling Mismatch (Low)
*   **Issue:** The Dashboard frontend was expecting `trade.entry_low` and `trade.entry_high` fields, but the backend `feature_store.py` was returning a grouped array `entry_zone: [low, high]`. This caused the frontend to display `$NaN - $NaN` (or blank).
*   **Status:** **FIXED.** Updated the serialization API to return distinct keys matching the React interface.

### E. Intraday Scalability (Architecture)
*   **Issue:** Currently, `daily_runner.py` is invoked once per day. It pulls Macro (FRED) and CFTC data, which only updates weekly/monthly. Running this every 5 minutes will immediately hit rate limits and cause blocking delays.
*   **Fix:** Implement `intraday_runner.py` which isolates the OHLCV feature vector calculations (using Yahoo Finance `interval=5m`) and skips the macro data pulls, applying the last known macro snapshot.

### F. Disconnected Header State & DB Failure (Critical)
*   **Issue:** The global `HeaderBar` UI component pulled the 'Regime' and 'Composite Score' from a WebSocket payload generated by a legacy, decoupled `multi_factor_engine`. Simultaneously, the `daily_runner` AI pipeline failed to persist actual HMM/LightGBM regime results to the DB due to a missing `symbol` argument in `save_regime_record()`. This left the entire dashboard rendering stale or "NEUTRAL" values.
*   **Status:** **FIXED.** Replaced the legacy WebSocket regime state with the centralized `_latest_intraday` RAM cache produced by the AI pipeline. Fixed the `save_regime_record` and `save_trade_recommendation` function signatures in `daily_runner.py`.

### G. Multi-Asset Independence Verification (Critical)
*   **Issue:** Audit required to verify WTI is not acting as a global proxy leaking into other commodities.
*   **Status:** **PASSED.** The AI engine (`walk_forward.py`, `RegimeEngine`, `SpreadModel`) dynamically routes models based on `symbol` arguments and strictly utilizes the local `feature_matrix` passed via the execution pipeline.

### H. Feature Source & Data Freshness Audit (Critical)
*   **Issue:** Audit required to detect mocked or stale features affecting live inferences.
*   **Status:** **PASSED (with warnings).** 95% of features (curve, technical, fundamental, macro) are successfully connected to Yahoo Finance or EIA. The `news_sentiment` feature defaults to 0.0 inside `feature_matrix.py` despite being fetched from FinBERT. `cftc_net_positions` relies on an unstable proxy endpoint.

### I. Structural Predictions Pipeline Failure (Critical)
*   **Issue:** Calendar Spreads and Butterflies are not being generated accurately for historical data. `PriceFetcher` synthesizes M2/M3 using lagging moving averages (`M1.rolling(21)`). Furthermore, `daily_runner.py` excludes spread symbols entirely, meaning no models are trained or executed for structural trades.
*   **Status:** **PENDING FIX.** `SPREAD_FLY_CRACK_AUDIT.md` generated. Requires replacing synthetic logic and rewiring the daily pipeline.

### J. Paper Trading Engine Blindspots (Medium)
*   **Issue:** The paper trading publisher currently ignores structural forecasts (Spreads, Flies, Cracks, RV) and solely listens to outright directional futures recommendations.
*   **Status:** **PENDING FIX.** `PAPER_TRADING_INTEGRATION_AUDIT.md` generated. Requires building a Signal Ranking Engine to prioritize high-conviction spread trades over outrights.

## 3. Recommended Actions
1. Deploy the `intraday_runner.py` to decouple the slow daily macro fetches from the fast 5-minute price updates.
2. Complete the Paper Trading AI Signal Ranking Engine to prioritize structural trades.
3. Validate that the walk-forward evaluation accurately reflects the new zero-leakage transformation loop.
4. Integrate `news_sentiment` directly into `feature_matrix.py` to remove the 0.0 mocked fallback.
5. Implement proper historical forward curve fetching to fix synthetic calendar spreads.
